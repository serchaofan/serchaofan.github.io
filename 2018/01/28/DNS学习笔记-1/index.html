DNS全称Domain Name System域名解析服务，用于解析域名与IP地址对应关系。

DNS运行在UDP上，UDP端口号53，也会使用TCP的53端口，先会用UDP查找，若UDP查不到或请求大于512字节时才用TCP查找。同时，UDP进行名称解析，TCP进行区域解析。

功能
  • 正向解析：根据域名查找对应IP
  • 反向解析：根据IP查找对应域名
分类
  • 主服务器：在特定区域内具有唯一性的域名解析服务器
  • 从服务器：从主服务器获取域名解析信息并维护，以防主服务器宕机
  • 缓存服务器：向其他服务器查询域名解析信息，提高重复查询效率
域名结构
  • . 根域，管理一级域名
  • com|org|gov|cn等一级域名，管理二级域名
  • baidu|google等二级域名，管理三级域名，当国家为一级域名时，com等一级域名便会下降一级别，依次类推
  • 依次会有三级，四级（很少了吧）
  • www|ftp|mail等主机域名，为提供服务的主机名

DNS系统采用阶层式管理，上一级的服务器只记录下一层的主机名（服务器名）

域名解析方法
  • 递归：服务器收到请求时，若不能解析，则把请求转发到下一台服务器直到有一台解析成功。注：是收到请求的服务器去问，一个问下一个，最后解析完成后原路返回。
  • 迭代：服务器收到请求时，若不能解析，则按根域->一级域名->二级域名->三级域名依次询问，直到解析成功。注：是本地服务器去不断问。

DNS域名解析服务中最高效的是Bind。Bind的程序名叫named。

实验

环境

  • Centos7，内核3.10
  • 虚拟机网段：192.168.163.0/24
  • 虚拟机IP：192.168.163.102/24
  • 网关：192.168.163.254
  • 主机名：system3.example .com

DNS服务相关配置文件

  • /etc/named.conf 主配置文件
  • /etc/named.rfc1912.zones 区域配置文件

步骤

  1. 安装bind服务

    yum groupinstall DNS\ Server

  2. 开启named服务

    防火墙放行dns，rich rules放行UDP和TCP的53端口

    systemctl enable named.service

    firewall-cmd --add-service=dns --permanent

    firewall-cmd --permanent --add-rich-rule='rule family=ipv4 port port=53 protocol=udp protocol=tcp accept'

    firewall-cmd --reload

  3. 修改配置文件

    修改配置文件最好先做备份 cp -a XX XX.bak

    首先修改/etc/named.conf

然后修改/etc/named.rfc1912.zones

配置正向解析区域文件

named.conf中也能写区域配置，但为了安全和管理，将主配置和区域配置分开为两个文件

然后配置解析数据信息文件/var/named/example.com.zone

配置文件有模板，可复制/var/named/named.localhost并改名

最好将文件名改为域名.zone

至此基础配置完成，重启服务
