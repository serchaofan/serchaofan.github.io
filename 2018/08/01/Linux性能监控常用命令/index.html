top与htop

top

查看动态进程状态，默认每5秒刷新一次。

进入top视图后的操作

htop

默认没有安装，需要dnf install htop

可通过鼠标点击操作，h查看帮助

uptime,free和vmstat

uptime

获取主机运行时间，查询系统负载

负载率(load)，即特定时间长度内，cpu运行队列中的平均进程数(包括线程)，一般平均每分钟每核的进程数小于3都认为正常，大于5时负载已经非常高。Linux运行队列包括正在运行的、在等待的、处于可中断睡眠态（IO等待）的进程。若为多核CPU，还需要除以核数。

平均负载最佳值为1，意味着每个进程都能立刻访问CPU，并且没有丢失CPU周期

free

查看内存与swap分区使用状况，实际是从/proc/meminfo中读取数据的

vmstat

报告进程、内存、分页、块IO、中断、CPU活动信息，能够显示平均数据和实时样本。

  • 虚拟内存模式
  • 磁盘模式

mpstat与iostat

mpstat与iostat都在sysstat包中，若没有这两个命令，则需要安装dnf install sysstat

mpstat

用于报告在多处理器服务器上每个可用CPU的统计数据。

iostat

ps和pstree

ps

ps命令有两种风格：BSD和Unix。BSD格式的参数前不加-，Unix格式会在参数前加-

  • 查看所有进程
  • 显示用户进程
  • 显示指定进程
  • 通过cpu或内存占用对进程排序
  • 以树显示进程层级关系
  • 查看指定父进程下的所有子进程
  • 显示进程的线程
  • 指定要显示的列
  • 通过watch命令将ps变为实时查看器

pstree

查看进程树，常用选项
